01

Summary of Invention

This disclosure describes a user interface method and system in which a body of structured content — whether narrative prose, professional history, analytics data, or product information — is decomposed into discrete, sequentially navigable units called sentences or nodes. Each unit carries typed metadata that drives synchronized, ambient changes across the entire interface at the moment of activation.

The invention is distinguished from conventional content scrolling, pagination, carousels, or slideshow interfaces by three interlocking properties:

1. Unit-Level Metadata Drives Global State

Each content unit carries a typed payload — chapter index, visual theme identifier, associated image reference, section classification, and optional structured data. Activating a unit triggers ambient state transitions across the full viewport: background palette shifts, panel content updates, image layer transitions, and navigation widget state updates all resolve in synchrony from a single unit activation event.

2. Sequential Navigation as the Primary Reading Contract

The interface does not scroll. Content is consumed one unit at a time, in forward or reverse sequence, via any input modality — scroll wheel, keyboard, swipe gesture, direct tap, or automated flow. The reading pace is the reader's own; the reading order is the author's. This contract is the format's argument.

3. Content-Type Agnosticism

The identical engine, interaction model, and visual system has been implemented across categorically distinct content domains: literary narrative, professional biography, client analytics reporting, hospitality branding, retail commerce, real estate presentation, and event invitation. The system is a medium, not a template.

The interface is the argument. The format is not chosen to present the content — it is chosen because it is the content's correct form.

02

The Sentence Engine

The sentence engine is the core navigable structure of the system. At render time, content is parsed into an ordered array of DOM elements — the navigable set. Each element in the set carries typed metadata via data-* attributes that define its role, its chapter membership, its associated image index, and any structured payload data.

Unit Types

The system defines the following unit types, each with distinct rendering and behavioral properties: poster (large display typography, primary visual impact), body (narrative or informational prose), pull (extracted emphasis, left-bordered, metric-capable), spec (key-value data row, scannable, hoverable), label (section marker, accent-colored, low opacity), break (structural separator), greeting (addressee or opener), and remark-thread (embedded annotation collection).

Active/Revealed/Past State Model

At any moment, exactly one unit in the navigable set is active — rendered at full opacity with ambient state reflecting its metadata. Units that have previously been active are past — rendered at reduced opacity, slightly more visible than revealed units that are ahead of the active position. This three-state opacity model provides spatial context for reading position without requiring a visible scroll position indicator.

Navigation Resolution

Navigation events — from any input modality — resolve to a target index in the navigable set. The engine applies the active state to the target unit, updates the past/revealed states of all other units, triggers the ambient state resolution pipeline, and scrolls the active unit into centered viewport position. All state updates are batched to a single animation frame.

Chapter Membership

Each unit carries a data-ch integer that assigns it to a chapter. Chapter membership is the primary axis of the ambient state system — background palette, HUD color scheme, panel content, and chapter widget state all resolve from the active unit's chapter value. The chapter set is sparse: only chapter transitions trigger full ambient re-theming, while intra-chapter navigation produces only unit-level state changes.

03

Ambient State Architecture

The ambient state system synchronizes the full visual environment of the interface to the active content unit. It operates on a CSS custom property token architecture, enabling smooth transitions across all affected elements simultaneously from a single JavaScript state change.

Chapter-Keyed Color Palettes

Each chapter in the system is assigned a complete color palette, expressed as a set of CSS custom property overrides applied to the body element via a chapter class (e.g. body.ch-3). The palette defines background color, three ink levels (primary, secondary, tertiary), accent color, accent-transparent, and border color. When the active unit's chapter value changes, the body class updates and all palette-consuming elements transition simultaneously via a shared transition-duration of 1.6–1.8 seconds.

HUD Palette Inversion

The navigation HUD (a floating widget that provides prev/next controls, section navigation, and interest triggers) maintains an inverted palette relative to the main viewport — light-on-dark when the main interface is dark, with chapter-keyed warm/cool tinting. This is achieved via a parallel set of --hud-* custom properties that also update on chapter transition, maintaining the HUD's legibility and visual distinction across all chapter palette states.

Image Layer Synchronization

A fixed full-viewport image layer behind the content stream carries a set of pre-loaded images indexed to match content unit data-img values. When the active unit's image index changes, the current image fades out and the indexed image fades in, with a simultaneous slow-scale transition (transform from 1.03 to 1.0 over 8 seconds) that produces an ambient parallax effect. The image layer is masked by a gradient overlay that preserves text legibility at all image states.

Panel Content Synchronization

The right-rail panel displays contextual information — product listings, portfolio entries, analytics stats, or biographical timeline — that updates as the reader moves through chapter-level transitions. The panel reflects the current chapter's subject matter, converting the right column from a static sidebar into an ambient companion to the stream.

Progress Indicator

A 2px fixed bar at the bottom of the viewport reflects the reader's position through the full navigable set as a percentage. The bar uses the active chapter's accent color, participating in the ambient palette system. This is the only positional indicator in the interface — there is no scrollbar.

04

Ghost Overlay System

The ghost overlay is a floating detail card that surfaces structured secondary data associated with specific content units. It is a core innovation of the system's data presentation mode — enabling dense information to remain legible in the stream while making its full detail available on demand, without modals, drawers, or page navigation.

Inline JSON Payloads

Hoverable content units carry structured data directly in their markup via a data-ghost attribute containing a JSON array of key-value pairs, each with an optional color-class designator (up, dn, or neutral). This architecture keeps the data co-located with the content unit that references it, eliminating the need for a separate data layer or API call for detail rendering.

Exclusion Zone Positioning

The ghost overlay uses a computed exclusion zone system to determine its resting position. At initialization, the system calculates bounding rectangles for all persistent UI elements — the navigation HUD, the right-rail panel, the sticky header, and the chapter widget — and stores them as exclusion zones. The ghost's idle position is computed by testing a priority-ordered set of candidate positions (above the trigger, below the trigger, left-flush of the stream column) against the exclusion zones, selecting the first non-overlapping candidate. This ensures the ghost never obscures interactive UI elements regardless of scroll position or viewport size.

Cursor-Following with Zone Transition

When the user's cursor is in the left half of the stream column, the ghost follows the cursor via a lerp-interpolated animation loop (follow factor 0.11). When the cursor crosses the stream midpoint toward the right, the ghost transitions to its computed idle position via a slower lerp (idle factor 0.07), drifting magnetically to clear space. This zone-based behavioral split means the ghost is always simultaneously accessible (following the cursor for precision reading) and unobtrusive (self-parking when the user approaches other UI).

Panel Stat Cross-Trigger

Ghost overlays can be triggered not only by stream content units but also by stat cards in the right-rail panel. When a panel stat is hovered, the ghost anchors into the stream column area — specifically to the right side of the stream at the vertical position of the triggering stat — establishing a visual bridge between the panel and the stream without the ghost intruding into the panel itself.

05

Context-Aware Interaction Surfaces

The system includes interaction surfaces whose content and behavior adapt dynamically to the reader's current position in the navigable set — specifically their current chapter and active sentence. These surfaces are not static UI components; they are responsive extensions of the reading state.

The Interest Panel

A collapsible panel anchored to the navigation HUD presents a set of context-specific interaction options — rendered as selectable pill buttons — whose labels and prompt text are drawn from a chapter-indexed lookup table. When the panel opens, it reads the active unit's chapter value and renders the corresponding pill set. If the reader navigates to a new chapter while the panel is open, the pill set and prompt text update to reflect the new chapter context. The panel surfaces the current sentence text as a reading anchor, attributing the contact initiation to a specific moment in the reading. On submission, the selected pill values and anchored sentence text are incorporated into a pre-composed email draft addressed to the author.

Chapter Widget

A floating chapter navigation widget, positioned at the top-right of the viewport, displays the current chapter name and provides a dropdown of all chapters with their ordinal numbers. The active chapter item updates in real time as the reader moves through the stream. The widget is independently draggable — its position is set by the user and persists across open/close cycles for the session. On mobile, the widget collapses to a compact bar.

Header Overlay

The sticky header reveals a four-column information overlay on hover — displaying contextual data relevant to the current document's subject (account details, recent activity, product listings, or career summary depending on implementation context). This overlay is not a navigation element; it is ambient detail that rewards curiosity without interrupting the reading state.

Narrative Toasts

The system supports author-embedded toast messages anchored to specific content units, which fire each time that unit is activated. These are not system notifications — they are part of the authored piece, a second voice that speaks alongside the sentence. Toast content can carry action buttons that drive navigation or external links.

06

The Dispatch Application

The Dispatch is an application of the sentence-progressive interface to client analytics reporting. It demonstrates that the identical engine used for literary narrative and professional biography is equally valid as a medium for presenting structured business data — and that the presentation mode itself transforms the reader's relationship to the data.

Analytics as Authored Prose

Monthly performance data — visit counts, session durations, chapter completion rates, traffic sources, remark counts — is authored as navigable prose using the full sentence type vocabulary. A pull node renders a large metric with delta indicator. A spec node renders a scannable key-value row with color-coded performance signals. A body node provides editorial context for the data. The reader navigates the report the same way they navigate a narrative — one sentence at a time, in sequence, the data arriving with considered editorial framing.

Hoverable Data Nodes

Spec nodes in the Dispatch carry data-ghost JSON payloads with full metric breakdowns — sub-metrics, month-over-month comparisons, segment splits — surfaced via the ghost overlay system on hover. The stream renders summary data; full detail is available on demand without leaving the reading flow. Panel stat cards trigger the same ghost overlay, projecting detail into the stream column when the panel is hovered.

Sections as Chapters

Report sections (Overview, Engagement, Remarks, Products, Looking Ahead) map directly to the chapter system — each with its own chapter-keyed palette, its own panel context, and its own section-widget label. The same ambient re-theming that moves a reader from Early Days to Agency Life in a biographical context moves a client from Overview to Engagement in a reporting context.

Live Sparkline and Bar Charts

The right-rail panel in the Dispatch implementation renders a JavaScript-generated sparkline (daily traffic over the reporting period) and a set of animated bar charts (chapter reach percentages) using only the native Canvas API and DOM — no charting library. These components participate in the panel's hover-inversion palette system, converting from dark to light when the panel is hovered.

The Dispatch does not present analytics. It presents a story about performance — authored with the same care as the platform it describes. Data delivered as dispatch.

07

Professional:Social Application

The sentence-progressive interface constitutes a direct reimagination of the professional social profile — most commonly exemplified by LinkedIn — as a narrative medium rather than a data entry form. This application is fully realized in the implementation referred to herein as link.html, subtitled "Open to Work."

The Core Argument

A conventional professional profile forces every person into an identical grid: job title, company, tenure, bullet points. The sentence-progressive format allows a career to be presented as it actually was — a sequence of choices, observations, environments, and consequences, told in the author's voice, navigated at the reader's pace. The format is not decorative. It is epistemological: it argues that a person cannot be adequately represented by a form.

Career Chapters as Chapter Palettes

Each era of a professional history maps to a named chapter with its own color palette. Early career, agency years, large-agency tenure, and independent work each carry their own ambient visual identity — warm amber for origin, slate indigo for urban agency, cool steel for corporate scale, rose for the present. The reader does not just read about these eras; they inhabit them visually as they move through the narrative.

Context-Aware Contact Initiation

The Interest Panel in this implementation reads the reader's current chapter and presents pill options calibrated to that career era — "The commute story," "WCAG / Pharma," "Cards Against Bad UX" for the NYC agency chapter; "A product," "A design system," "Let's just talk" for the Building chapter. The contact email pre-populates with the selected pills and the sentence the reader was on when they opened the panel. The hiring manager or collaborator who reaches out does so anchored to the specific moment in the narrative that resonated.

Grid Overlay with Live Portfolio Iframes

A grid overlay, triggered from the navigation HUD, renders all referenced web products as live iframe previews in a scaled tile layout — each tile a 4:3 viewport-scaled rendering of the actual live site. Clicking any tile opens the site in a new tab. This transforms the portfolio section from a list of links into a visual gallery of living products.

Platform Extension

The Professional:Social application extends the wolfie.life platform as a named vertical — wolfie.link — offering the sentence-progressive professional profile as a product to individuals who believe their career cannot be accurately represented by a form. The same design system, the same engine, the same conviction.

Multi-Vertical Platform Architecture

The full system is implemented as a multi-vertical platform operating under the Wolfgang & Co studio identity at wolfie.life. Each vertical is a domain-specific application of the identical sentence-progressive engine and design system to a distinct content category.

wolfie.love — Platform & Identity

The origin implementation. A literary memorial narrative for a dog named Wolfgang. Establishes the full engine vocabulary: sentence navigation, chapter-keyed palettes, image layer, ghost overlay, flow mode, visitor remarks, interstitial dark moment, narrative toasts, and all view modes. The platform's own proof of concept.

wolfie.food — Hospitality

Restaurant and hospitality digital presence. The sentence-progressive format applied to a restaurant's story — its ethos, its dishes, its welcome. Live menus. Operator dashboards. The Dispatch as monthly performance report.

wolfie.shop — Commerce

Retail and maker commerce presentation. Products authored as sentences. The store as editorial — the difference between a decision and a transaction surfaced through format alone.

wolfie.real — Real Estate

Property presence for independent brokers and boutique developers. A home presented as it feels, not as it measures. The sentence-progressive format applied to the most physical of digital representations.

wolfie.invite — Events

Digital invitation experience for occasions that matter. Weddings, openings, private gatherings. The invitation as the first act of the event — authored, navigated, remembered.

wolfie.link — Professional:Social

Career narrative as sentence-progressive interface. The professional profile reimagined as a story rather than a form. Context-aware contact initiation. Live portfolio grid. Twenty years of scar tissue, told with conviction.

Shared Design System

All verticals share the identical typographic system (Space Mono + Barlow Condensed), token architecture (CSS custom property palettes), interaction model (sentence engine, HUD widget, chapter widget, ghost overlay), and component vocabulary (spec nodes, pull nodes, poster nodes, interest panel). A product built on wolfie.life is recognizably wolfie.life regardless of its content domain.

09

Prior Art Distinction

The following distinctions are made from existing known systems and interaction paradigms.

vs. Scroll-Based Storytelling (e.g. NYT Snowfall, Scrollytelling)

Scroll-based narrative interfaces advance content via continuous scroll position. The sentence-progressive system does not scroll — each navigation event is a discrete state transition. Scroll position is not the navigation primitive; sentence index is. The interface does not degrade to a scrollable page.

vs. Presentation Software (PowerPoint, Keynote, Slides)

Slide-based systems navigate between full-screen frames, each replacing the previous. The sentence-progressive system maintains all units in the DOM simultaneously — the stream is always present, with past and future units visible at reduced opacity. The reading context is spatial, not sequential-replacement.

vs. E-Reader / Digital Book Interfaces

E-reader systems paginate text into page-sized blocks and navigate between pages. The sentence-progressive system navigates individual semantic units, each of which may be a single clause, a metric, a specification row, or a structural break. The unit granularity is authorial and semantic, not typographic.

vs. Dashboard / Analytics Interfaces

Conventional analytics dashboards present data in simultaneous multi-chart layouts for parallel visual comparison. The Dispatch implementation presents analytics sequentially, as authored prose with hoverable detail overlays. The reading mode is linear and editorial, not spatial and comparative.

vs. Professional Profile Platforms (LinkedIn et al.)

LinkedIn and analogous platforms represent professional identity as structured form data — job titles, tenures, bullet-pointed accomplishments — rendered in a standardized grid identical for all users. The wolfie.link implementation represents professional identity as sequential narrative with chapter-keyed ambient states, context-aware contact initiation, and authored editorial voice. The format is not customization of a template; it is a categorically different reading experience.

The novelty is not in any single component. It is in the system: unit-level metadata driving full-viewport ambient state, across categorically distinct content domains, via an identical engine.

Core Claims

The following claims describe the novel aspects of the invention as implemented. They are written for comprehension by a technically literate reader and to support subsequent formal patent claim drafting.

Claim 1 — Sentence-Progressive Navigation

A method of rendering structured content as an ordered set of discrete DOM elements, each assigned a sequential index, wherein user navigation events advance or retreat the active index by one position, the active element is rendered at full opacity and scrolled to a centered viewport position, and all other elements are rendered at reduced opacity according to their position relative to the active index.

Claim 2 — Unit-Metadata-Driven Ambient State

A method wherein each navigable content unit carries a chapter membership value, and wherein a change in the active unit's chapter membership triggers a synchronized update of a complete set of CSS custom property tokens applied to the root element, producing simultaneous ambient transitions of background color, text color hierarchy, accent color, border color, and HUD palette across all affected viewport regions, with a shared transition duration.

Claim 3 — Exclusion-Zone Ghost Overlay

A method of positioning a floating detail overlay wherein the overlay's idle position is computed at runtime by testing a priority-ordered set of candidate positions against a set of bounding rectangles derived from persistent UI elements, selecting the first candidate position that does not intersect any exclusion zone, and wherein the overlay transitions between cursor-following behavior and computed idle position based on the cursor's horizontal position relative to a computed midpoint of the content stream column.

Claim 4 — Inline Structured Data Payloads

A system wherein hoverable content units carry structured secondary data as inline JSON encoded in a data-ghost HTML attribute, said data comprising an array of key-value pairs with optional visual classification designators, rendered on demand in the ghost overlay without external data fetching or state management beyond the DOM element itself.

Claim 5 — Context-Aware Contact Initiation

A method wherein an interaction surface presents a set of selectable options whose labels and prompt text are drawn from a lookup table indexed by the active content unit's chapter membership value, and wherein the surface also captures the text content of the active content unit at the time of opening, incorporating both the selected options and the captured text into a pre-composed outbound communication addressed to the content author.

Claim 6 — Cross-Domain Engine Application

A platform comprising multiple distinct product verticals — including literary narrative, professional biography, client analytics reporting, hospitality branding, retail commerce, real estate presentation, and event invitation — all implemented using the identical sentence-progressive navigation engine, design token system, and interaction component vocabulary described in Claims 1–5, such that a reader who has encountered any one vertical will immediately recognize and be fluent in any other.

Claim 7 — Analytics-as-Authored-Prose (Dispatch)

A method of presenting periodic business analytics wherein quantitative data — visit counts, session durations, conversion metrics, segment distributions — is authored as a sequentially navigable set of typed content units including prose sentences providing editorial context, pull nodes rendering large metrics with delta indicators, and spec nodes rendering scannable key-value data rows with inline hover-activated detail overlays, delivered within the sentence-progressive interface system.
