Syntax Reference
Input types
plain sentence
Conversational. Think out loud.
Model responds in kind — narrative, exploratory
[pull] question
Isolate the one thing. Distill.
Model gives you the single truest answer — no padding
[spec] key :: value
Structured. Factual. No narrative.
Model responds as structured data — spec rows
[poster] topic
Demand a declarative headline.
Model answers with a bold, committed statement
[aside] detail
Parenthetical. Scrutinized detail.
Model treats this quietly — brief, footnote-level
[list] topic
Enumerate without prose.
Model responds as a clean list — no filler sentences
Inline modifiers
(parenthetical)
Embed a quiet aside in any sentence.
Model addresses this briefly, separately
—
Break. New thread of thought.
Resets context within the session
About
Syntax isn't formatting. It's intent declaration — you tell the model how to think, not how to display.

One bracket doing the work of three sentences of prompt engineering.
